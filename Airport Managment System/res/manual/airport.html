Airport

Before performing any other operations you will require to load an airport configuration within the application. Airport configurations can be loaded in a variety of ways:

Creating Airport

If you are a new user, the first action you will perform is creating a new airport configuration to use.

To create a custom airport click File->New->Airport.

You must specify both airports name and airports code and add at least one runway.

Read more about how to add a runway in the Runway tab.

Editing Airport

Once you have created an airport configuration, you may realise that some of the details you have entered are not correct. In such an event, you can easily edit the airport configuration at any time.

To edit an airport click Edit->Airport.

The GUI allows to change airports name, code and edit, add or remove runways.

Exporting an Airport

Now that your airport configuration configuration is correct, it is a good idea to save this configuration to a file such that it can be imported at a later date.

To export an airport click File->Export->Airport and select the directory where you want to save the airport xml file.

All data is saved in XML format.

Importing Airport

The application supports the ability to load a previously exported airport configuration from a file. This is useful as it avoids having to input the airport parameters each time you use the application.

To import an Airport from a predefined list of airports click File->Import->Airport and select it from the predefined list. (the selected file must end with extension .xml)
